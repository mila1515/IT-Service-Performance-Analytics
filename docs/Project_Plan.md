# Project Plan

## Phase 1 - Business Understanding

Define the business problem, objectives, stakeholders and expected outcomes.

## Phase 2 - Data Understanding

Load the raw incident dataset, inspect its structure and identify initial data quality issues.

## Phase 3 - Data Cleaning

Prepare the dataset for analysis by handling missing values, data types, duplicates and derived fields.

## Phase 4 - Exploratory Data Analysis

Analyze incident volumes, SLA compliance, resolution times and operational patterns.

## Phase 5 - SQL Data Modeling

Create a structured SQL layer to support reusable analysis and dashboarding.

## Phase 6 - Power BI Dashboard

Build an interactive dashboard focused on SLA monitoring, team performance and incident drivers.

## Phase 7 - Business Recommendations

Translate analytical findings into operational recommendations for IT management.

## Phase 8 - GitHub Documentation

Document the project clearly for portfolio review and recruiter evaluation.
